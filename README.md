# Escopo do Problema
Usuário possui uma empresa de venda de joias e está com muita dificuldade em acompanhar o pagamento dos produtos realizados pelos seus clientes, assim como também o acompanhamento dos produtos comprados mantidos em estoque.

Há um acompanhamento dos produtos vendidos por meio de fichários, cada um contendo as informações do pagador e do produto comprado. Dado o contexto principal do serviço que o usuário realiza, o mesmo gostaria de possuir um sistema para gerenciamento das vendas realizadas e controle do estoque de produtos. Além disso, o usuário quer um sistema à parte que servirá somente para o seu cliente acompanhar as compras realizadas.

Dado o contexto, serão 3 sistemas, dois deles integrados em um sistema completo, dedicado ao proprietário e seus funcionários. Por fim o terceiro dedicado exclusivamente para o cliente comprador.

# Objetivo do Projeto
A combinação de cada entidade presente no sistema devem funcionar em harmonia, o cliente deve realizar o acompanhamento e compras no sistema. O proprietário fica responsável pela definição de papéis para seus funcionários e acompanhamento das vendas dentro do sistema. O funcionário é responsável pelo estoque, à qual conterá todos os itens da empresa em que o próprio funcioário fará seu cadastro
